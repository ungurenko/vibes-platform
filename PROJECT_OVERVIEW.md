# VIBES - Платформа Вайб-Кодинга

## Описание проекта

**VIBES** - это образовательная платформа для обучения созданию веб-сайтов и веб-сервисов с помощью искусственного интеллекта (AI-assisted coding). Проект представляет собой полноценную LMS (Learning Management System) с административной панелью, системой трекинга прогресса и встроенным AI-ассистентом.

**Целевая аудитория:** Новички без опыта программирования, желающие научиться создавать сайты через промпт-инжиниринг и работу с нейросетями.

**Основная концепция:** "Вайб-кодинг" - методология создания веб-проектов через текстовые запросы к AI вместо написания кода вручную, с акцентом на дизайн-эстетику и современные технологии.

## Технологический стек

### Frontend
- **React 19.2.1** - основной UI фреймворк
- **TypeScript 5.8.2** - статическая типизация
- **Tailwind CSS** (via CDN) - utility-first CSS фреймворк
- **Framer Motion 12.23** - анимации и transitions
- **Lucide React 0.555** - библиотека иконок
- **React Markdown 9** - рендеринг Markdown контента
- **canvas-confetti** - эффекты конфетти при достижениях

### Build & Development
- **Vite 6.2.0** - быстрый сборщик и dev-сервер
- **@vitejs/plugin-react** - React плагин для Vite
- **ESLint** - линтер кода

### Backend & Infrastructure
- **Supabase** (@supabase/supabase-js 2.89.0) - Backend-as-a-Service
  - PostgreSQL база данных
  - Аутентификация пользователей (email/password)
  - Real-time подписки
  - Storage для файлов
- **Vercel** - хостинг и serverless функции
- **OpenRouter API** - прокси для доступа к различным LLM моделям

### AI Integration
- AI-ассистент через OpenRouter API
- Поддержка различных моделей (Claude, GPT и др.)
- Serverless функция `/api/chat.js` для безопасного проксирования

### Архитектурные слои

1. **Презентационный слой** (Views)
   - 18 компонентов-страниц с разделением на студенческие и административные
   - Адаптивный дизайн для мобильных и десктопных устройств

2. **Бизнес-логика** (App.tsx)
   - Управление глобальным состоянием через useState/useEffect
   - Client-side routing через состояние activeTab
   - Управление темой (dark/light mode)
   - Управление режимом (student/admin)
   - Интеграция с Supabase для аутентификации

3. **Data Layer**
   - **Supabase PostgreSQL** - основное хранилище данных
   - **LocalStorage** - кеширование сессий и настроек пользователя
   - **data.ts** - статические справочные данные

4. **API Layer**
   - `/api/chat.js` - serverless функция для AI-ассистента
   - `lib/supabase.ts` - обертки для работы с Supabase

5. **Shared Components**
   - Переиспользуемые UI-компоненты
   - Глобальные контексты (SoundContext)

### Используемые паттерны
- Context API для глобального состояния
- Component Composition
- Custom Hooks (useSound)
- Props Drilling для навигации
- Graceful Degradation (fallback на моки при отсутствии Supabase)

## Для AI-агентов

При работе с этим проектом учитывайте:

1. **Основной код находится в:**
   - `App.tsx` - главная логика приложения (500+ строк)
   - `views/` - все страницы приложения
   - `data.ts` - статические данные (49 КБ моков)
   - `types.ts` - TypeScript определения

2. **При изменении UI:**
   - Используйте Tailwind CSS классы
   - Следуйте существующим паттернам дизайна
   - Проверяйте мобильную адаптивность
   - Добавляйте анимации через Framer Motion

3. **При работе с данными:**
   - Проверяйте наличие Supabase подключения
   - Обеспечивайте fallback на моки из data.ts
   - Обновляйте TypeScript типы в types.ts
   - Синхронизируйте изменения между admin и student views

4. **При добавлении функционала:**
   - Следуйте существующей архитектуре
   - Добавляйте новые views в директорию views/
   - Обновляйте навигацию в Sidebar.tsx
   - Регистрируйте routes в App.tsx

5. **Тестирование:**
   - Проверяйте оба режима (student/admin)
   - Тестируйте обе темы (light/dark)
   - Проверяйте работу без Supabase (fallback mode)
   - Тестируйте на мобильных устройствах

6. **Деплой:**
   - Изменения автоматически деплоятся через Vercel
   - Проверяйте environment variables в Vercel
   - Тестируйте production build локально через `npm run preview`
